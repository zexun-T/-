Vue.js 学习笔记

什么是渐进式？

Vue.js（读音 /vjuː/, 类似于 view） 是一套构建用户界面的渐进式框架。渐进式也就是“Progressive”，按照我的理解，渐进就是一步一步的意思，大概的意思应该是框架你可以只用到一部分，而不用把所有的东西都用上。

vue采用的是分离的状态：

这也就是说，当你的项目比较小比较简单的时候，我们就可以只用核心部分，也就是图中的“Declarative Rendering”（声明式渲染），跟“Component System ”（组件系统），而后按需加载加载其他部分

这就是我对渐进式框架的理解，优点在于降低了学习复杂度，也让框架变得比较轻。

vue的两个核心

1、响应式的数据绑定

Vue 的核心库只关注视图层 以往的数据交互，是我们通过Ajax向服务器请求数据，然后我们通过获取操作DOM节点来改变视图，每次交互数据改变时，又要重新操作，而Vue让我们减去了操作DOM的过程，当数据发生改变时，自动帮我们更新视图。Vue还会双向绑定数据，当你在视图层操作发生改变时，监听变化从而改变其数据内容。（利用了MVVM的思想）

2、视图组件化

这是一个组件树，通过多个小组件拼接，组成一个大组件，把组件封装起来就形成一个页面。而改变一个小组件，就不会影响其他组件页面。像堆积木一样，互相引用从而组装起来。组件拿出来别的页面也可以使用，实现组件的重复使用，有利于维护。

开始使用vue

1、创建实例：(需要已安装好vue.js，并将vue.js引入)

<div id='demo'>{{name}}</div>
<script>
    var vm = new Vue({
        el: '#demo',   //id选择
        data: {
            name: "路飞"
        }
    })
</script>

Vue是响应式的，当你在浏览器后台修改vm.name的值时，你会看到页面会相对应更新

2、绑定元素属性(attribute)

v-bind
```html
<div id='demo'>
    <span v-bind:id='id' v-bind:class='[myClass1,myClass2]'>{{name}}</span>
</div>
<script>
    var vm = new Vue({
        el: '#demo',   //id选择
        data: {
            name: "路飞",
            id:"only",
            myClass1:"demo1",
            myClass2:"demo2"
        }
    })
</script>

还可以给属性添加上布尔值，当true的时候，添加上该属性值，当false的时候，该属性值消失。

<div id='demo'>
    <span v-bind:class='{demo1:ifClass}'>{{name}}</span>
</div>
<script>
    var vm = new Vue({
        el: '#demo',   //id选择
        data: {
            name: "路飞",
            ifClass:true  //当false的时候，该属性值消失
        }
    })
</script>
v-if 跟 v-show
```html
<div id='demo'>
    <span v-if="ifseen">{{name}}</span>
</div>
<script>
    var vm = new Vue({
        el: '#demo',   //id选择
        data: {
            name: "路飞",
            ifseen:true  //false的时候，直接删除DOM节点
        }
    })
</script>
```html
<div id='demo'>
    <span v-show="ifseen">{{name}}</span>
</div>
<script>
    var vm = new Vue({
        el: '#demo',   //id选择
        data: {
            name: "路飞",
            ifseen:true  //false的时候，设置display:none;
        }
    })
</script>

如果频繁需要显示消失功能，那么就采用v-show的方式，而v-if是直接通过添加删除子节点的方法，会进行重排重绘，耗费性能

如果不频繁的话，使用v-if更好，因为false时没有DOM节点，初始化时不用加载

v-for
```html
<ul id="only">
    <li v-for ="item to list">{{item.fruit}}</li>  <!--item遍历数组list的值-->
</ul>
<script>
    var vm = new Vue({
        el: '#only',   //id选择
        data: {
            list:[{fruit:"苹果"},{fruit:"香蕉"},{fruit:"橙子"}]
        }
    })
</script>
v-on
```v-on:click


<div id='demo'>
    <span>{{name}}</span>
    <button v-on:click = "cli">点击</button>
</div>
<script>
    var vm = new Vue({
        el: '#demo',   //id选择
        data:{
        	name:'路飞'
    	},
        methods:{                        //创建methods对象，定义触发事件函数
            cli:function(){
                vm.name = '鸣人'
            }
        }
    })
</script>


```v-on:keyup.enter```表示只有按起回车键，才会执行事件

```v-model```用来显示input输入框中的初始值
```html
<div id='demo'>
    <span>{{message}}</span>
    <input type="text" v-model="message" v-on:keyup="keyup" style="height:30px">
</div>
<script>
    var vm = new Vue({
        el: '#demo',
        data: {
            message: "Hello,world!"
        },
        methods:{
            keyup:function(){
                vm.message = this.value,
                console.log(this.value)
            }
        }
    })
</script>

3、生命周期

生命周期钩子

生命周期钩子的函数，是用于不同阶段添加自己的代码的机会

挂载：我理解的挂载，是通过new Vue()语句创建的对象后，没有执行没有渲染，而是放到HTML文档中，这个过程就是挂载，我也找不到权威的解释，这样一来可能解释得通，通过Ajax请求出来数据后，把数据插进去，再一起挂载到HTML文档中执行渲染，此时DOM元素被vm.$el替换

生命周期钩子函数 作用时间顺序
beforeCreate : function(…){…} 实例初始化之后执行的函数，即整个页面创建之前执行
created : function(…){…} 实例初始化数据（data数据绑定，v-on事件绑定）之后，调用的函数，此时挂载阶段还没开始
一般函数内容为Ajax请求
beforeMount : function(…){…} 在挂载开始之前被调用，相关渲染函数首次被调用
Mounted : function(…){…} 挂载成功后被调用
DOM元素被vm.$el替换
实例激活，监听beforeUpdate跟update
beforeUpdate : function(…){…} 数据更新时调用
update : function(…){…} DOM已经更新，组件更新完毕后调用
