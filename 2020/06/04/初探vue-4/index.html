事件绑定v-on

简单的处理逻辑，可以直接写在值上面：

<button v-on:click="counter += 1">Add 1</button>

比较复杂的处理事件，应该调用方法，在methods 对象中定义方法，并且可以传参

<div id="example-1">
  <button v-on:click="say('hi')">Say hi</button>
  <button v-on:click="say('what')">Say what</button>
</div>
<script>
new Vue({
  el: '#example-1',
  methods: {
    say: function (message) {
      alert(message)
    }
  }
})
</script>

事件修饰符

名称 作用
.stop 组织事件冒泡
.pervent 组织元素默认行为（如form表单默认事件提交会跳转链接）
<form @submit.prevent></form>
.capture 组织捕获行为
.self 限制事件仅作用于节点本身
.once 事件只有效触发一次

.self跟.stop 的区别：

.self是将事件设置为只有自己本身触发的时候才触发，如果是冒泡传递上来的事件不触发自身事件

而使用.stop可以阻止事件的冒泡,这样在点击的时候不会触发外部的点击事件

名称 键值 对应按键

当需要多个按键才能实现操作的时候，多利用组合修饰符

系统组合修饰符 对应按键
.ctrl Ctrl键
.alt Alt键
.shif Shift键
.meta 系统菜单键 
<p @click.ctrl="withCtrl">do something</p>

这样单纯点击是不会触发withCtrl事件，只有配合ctrl+点击能触发

组件的注册

组件的注册可分为，全局注册跟局部注册

之前用的都是全局注册，昨天写了个对数组的排序跟筛选，然后发现sort( )函数会对原数组做出修改，于是我对该实例进行修改成组件注册，这样每次创建实例的时候，里面的数组都是原始值。就是将排序的函数单独摘出来写成一个组件。则有：

<div id='two'>
    <filter v-bind:numbers='firstNumbers'></filter>  <!--第一个函数为了排序，但是会改变原数组 -->
</div>
<div id="three">
    <filter v-bind:numbers='lastNumbers'></filter>  <!--重新new 一个组件，数组数据不会变 -->
</div>
<script>
    Vue.component('filter', {
        props: ['numbers'],   //prop向子组件template里面的模板赋值
        template: `<ul><li v-for="n in numbers">{{n}}</li></ul>`
        })
    new Vue({
            el: '#two',
            data: {
                numbers: [9, 4, 3, 1, 2, 5]
            },
            computed: {
                firstNumbers: function () {
                    return this.numbers.sort(function (a, b) {  //对原数组做出了改变
                        return a - b;
                    	})
                	}
        	})
    new Vue({
            el: '#three',
            data: {
                numbers: [9, 4, 3, 1, 2, 5]
            },
            computed: {
                lastNumbers: function () {
                    return this.numbers.filter(function (number) {   //不会对原数组做出改变。
                        return number <= 3
                    })
                }
            }
        })
</script>

局部注册

全局注册所有的组件意味着即便你已经不再使用一个组件了，它仍然会被包含在你最终的构建结果中，这样会让用户的下载量变多了，于是就有了局部注册，像声明对象一样声明一个组件

<script>
var ComponentA = {  ...  };
var ComponentB = {  ...  };
var ComponentC = {  ...  };
    //在 components 选项中定义你想要使用的组件：
new Vue({
  el: '#app',
  components: {
    'component-a': ComponentA,
    'component-b': ComponentB
  }
})
</script>

局部注册的组件在其子组件中不可用

如果你希望 ComponentA 在 ComponentB 中可用，则你需要这样写：

var ComponentB = {
  components: {
    'component-a': ComponentA
  }
